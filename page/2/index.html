jQuery中ajax get/post请求的语法

作用: 异步的动态网站交互应用技术 无刷新的情况下 实现数据的交互
jQuery提供Ajax封装
$.get() 以GET方式发起Ajax请求 只能发送get请求 只有4个参数
1 - url 必填项 规定将请求发送到那个URL
2 - data 可选 规定连同请求发送到服务器的数据 参数的意思
3 - success 可选规定当请求成功时运行的函数
4 - dataType 可选 规定预计的服务器响应的数据类型 默认的jQuery会智能判断
$.post() 以POST方式发起ajax请求 只能发送post请求 只有4个参数

jQuery中的ajax
$.ajax({}) 可配置的方式发起Ajax请求
1 - url 接口地址
2 - type 请求方式
3 - timeout 设定请求超时时间 就是设置一个默认请求的时间 超过时间 数据还没回来 就不要了 要是请求成功就响应success 不成功就响应error
4 - dataType 告诉底层$.ajax将返回来的数据以何种格式解析 人为的设置底层$.ajax用哪一种语法解析 支持 json xml jsonp
5 - data 发送请求数据 参数的意思
6 - beforeSend:function () { return false } 请求发起前调用 before：在…之前 send：发送 用于：在发送之前需要做的业务逻辑,用于页面中数据的判断 return false 会阻止请求的发送
7 - success 成功响应后调用的函数 一般用于渲染页面
8 - error 错误响应时调用
9 - compleat 响应完成时的响应的函数 无论成功还是失败 一般用于数据的初始化

布局思路

0 这种布局在所有的页面都是通用的 所以写在公用样式里面
1 头部和尾部定死位置
2 中间模块 高度 = 屏幕的高度 - 头部高度 - 尾部高度
中间模块里面的子元素高度 肯定高于中间这个大模块
3 html 和 body 高度：100%
4 body 加入 padding-top: 头部的高度; padding-bottom: 尾部的高度;
5 中间模块高度: 100%;

}

padding-top: 45px;
padding-bottom: 45px;
}

hexo博客教程


1-安装hexo前需要安装：Git node

2-进入 hexo官网：https://hexo.io/zh-cn/

3-在hexo官网首页中复制：npm install hexo-cli -g
注意/该代码是在git中执行/在自己喜欢的分盘或者桌面创建一个文件夹打开gitbash粘贴/执行后需要等待一段时间

4-再次执行：hexo init
执行完该文件后退出gitbash 因为已经在文件中生成了文件 在自己创建的文件中再次打开gitbash 执行: hexo s

5-安装next主题
进入next主题官网 下载next主题 下载下来后解压改名成next
把处理好的next包复制到hexo文件夹的themes目录中

6-启动next主题 http://theme-next.iissnan.com/
在hexo文件的根目录下 找到_config.yml文件打开该文件
ctrl+F 搜索 theme: 把这里的landscape 改成 next
在hexo的根目录下打开git 执行以下代码
$ hexo clear 清理缓存
$ hexo g 生成文件
$ hexo s 启用服务器 在我们需要打开hexo文件的时候 该文件不能关掉

7-设置hexo的语言-改成中文

打开hexo根目录底下的_config.yml文件
找到 languge: zh-Hans
再次打开gitbash执行：hexo s

8-开启hexo的标签导航

1-打开next文件根目录下的_config.yml
2-ctrl+F 搜索menu
3-把前面加#的符号都去掉 再回到hexo刷新页面

9-创建页面 - 修复打开标签分类标签时的报错

在hexo的根目录下打开gitbash
ctrl+c先把hexo服务器关闭
创建标签页: hexo n page tags
执行完之后 会在 source目录下多出一个文件: tags
打开该文件里的index.md文件
title: 标题
date: 时间
type: “tags” 这个必须要填 不然不会显示
hexo s 开启服务器 不开启服务器是无法打开hexo页面的

10-主题修改 - 在next的官方文档中 根据官方文档的要求来修改

在next的根目录下打开配置文件 _config.yml
ctrl+F 搜索关键字 scheme 喜欢那个主题就把前面的#号去掉就可以了
Muse - 默认 Scheme，这是 NexT 最初的版本，黑白主调，大量留白
