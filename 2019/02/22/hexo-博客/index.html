hexo博客教程


1-安装hexo前需要安装：Git node

2-进入 hexo官网：https://hexo.io/zh-cn/

3-在hexo官网首页中复制：npm install hexo-cli -g
注意/该代码是在git中执行/在自己喜欢的分盘或者桌面创建一个文件夹打开gitbash粘贴/执行后需要等待一段时间

4-再次执行：hexo init
执行完该文件后退出gitbash 因为已经在文件中生成了文件 在自己创建的文件中再次打开gitbash 执行: hexo s

5-安装next主题
进入next主题官网 下载next主题 下载下来后解压改名成next
把处理好的next包复制到hexo文件夹的themes目录中

6-启动next主题 http://theme-next.iissnan.com/
在hexo文件的根目录下 找到_config.yml文件打开该文件
ctrl+F 搜索 theme: 把这里的landscape 改成 next
在hexo的根目录下打开git 执行以下代码
$ hexo clear 清理缓存
$ hexo g 生成文件
$ hexo s 启用服务器 在我们需要打开hexo文件的时候 该文件不能关掉

7-设置hexo的语言-改成中文

打开hexo根目录底下的_config.yml文件
找到 languge: zh-Hans
再次打开gitbash执行：hexo s

8-开启hexo的标签导航

1-打开next文件根目录下的_config.yml
2-ctrl+F 搜索menu
3-把前面加#的符号都去掉 再回到hexo刷新页面

9-创建页面 - 修复打开标签分类标签时的报错

在hexo的根目录下打开gitbash
ctrl+c先把hexo服务器关闭
创建标签页: hexo n page tags
执行完之后 会在 source目录下多出一个文件: tags
打开该文件里的index.md文件
title: 标题
date: 时间
type: “tags” 这个必须要填 不然不会显示
hexo s 开启服务器 不开启服务器是无法打开hexo页面的

10-主题修改 - 在next的官方文档中 根据官方文档的要求来修改

在next的根目录下打开配置文件 _config.yml
ctrl+F 搜索关键字 scheme 喜欢那个主题就把前面的#号去掉就可以了
Muse - 默认 Scheme，这是 NexT 最初的版本，黑白主调，大量留白
Mist - Muse 的紧凑版本，整洁有序的单栏外观
Pisces - 双栏 Scheme，小家碧玉似的清新

11-设置头像

在next的根目录下 打开_config.yml配置文件
ctrl+F 搜索关键字 avatar
把avatar: 前面的#号去掉 这个images是存放图片的 你可以自己存图片进来 images在next的source目录下

12-设置 作者昵称 与及 网站标题

在hexo根目录下打开配置文件_config.yml
搜索找到 author: 作者昵称
title: 网站标题
修改完后 刷新hexo页面是无法看到改变的 需要生成文件
在hexo目录下打开git 执行以下代码
hexo clear 清除缓存
hexo g 生成文件
hexo s 开启服务器

13-设置 侧边栏社交链接

打开next文件中的_config.yml配置文件
搜索关键字: social
把social: 前面的#去掉
把github和email的#号去掉 设置自己的链接 email这个是微博 不是邮箱

14-友情链接

在next文件中打开配置文件 _config.yml
搜索： #links_layout: inline
把links: 前面的#去掉
Title: http:…. 也把友情链接的#也去掉 设置一下链接
title2： http:… 可以设置多个友情链接

15-新建文章

1 先关闭hexo服务器 - 在hexo的根目录下打开git - 执行hemo c 关闭服务器
2 创建文章: 在hexo的根目录下 打开git 输入命令行 - hexo n 我的文章 然后回车
3 开启服务器
4 然后刷新页面 就可以在首页看到多出来一篇文章

16-新建的文章 在标签和分类中显示出来

1 打开新建的文章.md
2 在date时间下 添加:
3 tags: 

- 导航  这两个东西是是标签页显示的东西
- 分享

4 categories: 前端 这个前端是在分类页面显示的东西
5 正文需要在 — 这三个横线下开始书写
6 #一级标题
7 ## 二级标题

18-开启文章的打赏

打开next的配置文件
ctrl+f 搜索 Reward
把下面三个的前面#号去掉
reward_comment: 你的支持就是我最大的动力
wechatpay: /images/…. 微信支付二维码
alipay: /images/…. 支付宝二维码

19-开启订阅公众号

打开next配置文件
搜索: wechat_subscriber:
把wechat enabled qcode description 前面的#去掉
qcode 是二维码图片
description 对这个订阅的描述: 扫一扫上面的二维码可以关注我哦

20-设置头像为园形鼠标移上头像可旋转

打开next根目录上的 source css _common components sidebar 打开sidebar-author.styl
在.site-author-image {
添加css样式: border-radius: 50%; transition: 2s all;
}

在给.site-author-image添加一个移动事件
.site-author-image:hover { transform:rotate(360deg); }

21-在首页设置阅读全文

打开next配置文件
搜索：auto_excerpt
enable: 把false改成true
length: 设置的高度

22-给主题添加动态背景

打开next配置文件
搜索：canvas
下面有四个false 这四个false代表四个不同的动态样式 喜欢那个样式就把那个的样式设置成true

23-给博客的左上角和右上角添加fork Github

打开 GitHub Corners网站: http://tholman.com/github-corners/
复制一个你喜欢的样式 链接
打开next 里面的layout 打开 _layout.swig文件
去到23行的

在这一行的下面添加刚刚复制的那个链接
把复制的那个链接GitHub地址改成自己的 href=”” 在这修改
回到首页刷新 没生效 就得去git ctrl+c 关闭服务器
hexo clear 清理缓存
hexo g 生成文件
hexo s 开启服务器

26-g把hexo部署到github上
1- 配置git-bash的用户名和邮件 - git config –global user.name xxx - git config –global user.email xxx
2- 在github上创建存储库
3- 在存储库上的存储库名称一定要和你的github登录名一样后面的github.io是大家都一样的 :

4- 然后点击创建存储库
5- 设置一个秘钥 - 在gitbash中输入: ssh-keygen -t rsa -C 'CGFeng-Email@qq.com‘ - 回车
然后在下面的指示中找到给出的文件位置 拿到秘钥
6- 在秘钥的指定地点打开 id_rsa.pub - 全选 -复制
7- 回到github中 打开设置 - 选择 SSH and GPG keys 创建秘钥 - 把刚刚复制到的秘钥粘贴到里面
8- 测试 本地的hexo和github是否关联成功: 打开gitbash - 输入 : ssh -T git@github.com 回车 - 按下面的执行 选择yes
然后就会提示：Hi CGFeng-Email! You’ve successfully authenticated, but GitHub does not provide shell access. 说明关联成功
9- 打开 hexo 配置文件 拉倒底部 - 添加
type: git
repo: git@github.com:CGFeng-Email/cgfeng-email.github.com.git 注意：这个是刚刚创建的存储库链接
branch: master 保存退出
10- 打开git-bash
输入:
hexo g 这是生成文件
hexo d 这是部署文件到github存储库 / 有些人这里会报错
如果 hexo d 报错 则 执行： $ npm install hexo-deployer-git –saveg

更新
hexo clear
hexo g
hexo d

26 更换电脑时 hexo的部署

1、在新电脑上用Git bash 陆续跑一遍以下镜像路径 在任意盘任意文件夹
$ npm install hexo-cli -g
$ hexo init blog
$ cd blog
$ npm install

$ hexo server

2、 把自己原有的hexo文件拉进新的电脑 

2.1: 在该hexo文件根目录下 打开git bash 输入 : ssh-keygen -t rsa -C 'CGFeng-Email@qq.com' 注意: 这里的邮箱地址 必须和当前的git bash设置的邮箱地址一样 然后根据给出的路径 在秘钥的指定地点打开 id_rsa.pub - 全选 -复制 
----
2.2: 回到github网站上 打开自己的博客库 -> 点击播客库的 settings 设置 -> 点击 deploy keys 部署秘钥 -> 点击 add deploy key 添加部署秘钥 -> 添加部署秘钥时 下面会有个 勾选 允许写访问是否 一定要勾上
2.3: 测试 本地的hexo和github是否关联成功: 打开gitbash - 输入 :  ssh -T git@github.com 回车 - 按下面的执行 选择yes

然后就会提示：Hi CGFeng-Email! You’ve successfully authenticated, but GitHub does not provide shell access. 说明关联成功
